# 要件: ルーラー表示機能の追加

対応 issue: #2

## 背景 / 課題

AGENTS.md の主要機能「ルーラー表示」が未実装。SEU では固定長フォーマットの桁位置が分かる
ルーラーが常時表示されるが、本拡張には該当コードが一切存在しない（`grep -rin "ruler" src/` で 0 件）。
VSCode で AS400 の固定長ソース（RPG/RPGLE/CLP/DDS 等）を編集する際、各カラムが何桁目で、
どのフィールド（Factor1 / OpCode / Result 等）に当たるかが分からず、SEU 利用者にとって編集効率が落ちる。

## 目的 / ゴール

VSCode 編集時に、フォーカスしている行の上側に固定長フォーマットの桁位置ルーラーを表示し、
SEU と同等の「桁位置が一目で分かる」編集体験を提供する。ソースは一切書き換えない。

## スコープ

### 対象

- フォーカス行の **上側** に、桁位置ルーラーをデコレーション（入力補助）として表示する。
- ルーラーは 2 段構成とし、表示モードを切り替えられる：
  - **目盛り段**：桁番号の目盛り（SEU 風。5 桁ごとに `+`、10 桁ごとに桁番号）。全ファイル共通。
  - **フィールド境界段**：フォーカス行のスペック種別（C/D/F/O/P/H-Spec, CL 等）を判定し、
    フィールド境界とラベル（Factor 1 / OpCode / Factor 2 / Result / Name / From / To 等）を表示。
    既存の `resources/navigation/*-keyword-columns.json` のカラム定義を流用する。
- 対象拡張子（`rpg`, `rpgle`, `clp`, `dds`, `dspf`, `prtf`, `cmd`）のファイルでのみ表示する。
  判定は既存の `fileScope.ts`（`isInScopeDocument`）に従う。
- **ステータスバーによる 3 状態トグル**：ステータスバーのクリックで以下を循環切替する。
  1. 非表示（OFF）
  2. ルーラーのみ（目盛り段のみ）
  3. ルーラー＋フィールド境界（目盛り段＋フィールド境界段）
  - 現在の状態がステータスバーの表示で分かること。

### 対象外

- RPGLE の自由記述（free format）対応（固定長フォーマットのみが本 PJ の方針）。
- SOSI（制御コード）表示そのものの変更（別機能。issue 別管理）。
- 対象拡張子の拡張（issue #4 で対応済み）。
- ルーラーの配色・スタイルの細かなユーザーカスタマイズ（最小限の妥当な既定で可。将来拡張）。
- 桁位置定義（フィールド境界）の値そのものの新規策定（既存 JSON 定義を正とする）。

## 機能要件

- フォーカス行が変わるたびに、その行の上側ルーラーを更新する。
- スペック種別を判定し、フィールド境界段の内容を種別に応じて切り替える。
  判定不能・空行・対象外行では目盛り段のみ表示（フォールバック）。
- 表示モード（3 状態）はステータスバークリックで循環し、即座に表示へ反映される。
- 表示モードの状態は適切に保持される（セッション内で維持。永続化の要否は spec で決定）。
- ソースの文字・桁を一切書き換えない（デコレーション/装飾でのみ実現）。

## 非機能要件 / 制約

- ソース非破壊（入力補助表示であること）。コピー＆ペーストやファイル保存に影響しないこと。
- 既存機能（SOSI 表示 `dbcsShiftMarkers`、タブナビゲーション等）と共存できること。
- フォーカス移動・編集時の更新で体感的な遅延が出ないこと。
- 既存の登録様式（`registration.ts` で各機能を register）に沿って組み込めること。
- フィールド境界はファイル種別ごとの桁位置定義を差し替え可能な設計が望ましい
  （既存 JSON 定義の利用を前提）。

## 完了条件 (受け入れ基準)

- [ ] 対象拡張子ファイルで、フォーカス行の上部にルーラー（目盛り段）が表示される。
- [ ] スペック種別（C/D/F/O/P/H-Spec, CL）に応じたフィールド境界段が表示される。
- [ ] ステータスバークリックで「非表示 → ルーラーのみ → ルーラー＋境界 → 非表示」と 3 状態が循環し、
      現在の状態がステータスバーで判別できる。
- [ ] 対象外拡張子のファイルではルーラーが表示されない。
- [ ] ソースの内容（文字・桁）が一切書き換わらない。
- [ ] 既存機能と併用しても表示が破綻しない。

## 未確定事項 / 確認したいこと

- 表示モードの状態をワークスペース設定として永続化するか、セッション内保持のみか（spec で決定）。
- 目盛り段／フィールド境界段の具体的なラベル文言・幅（狭いフィールドのラベル省略規則など）。
- DBCS（SOSI）を含む行で見かけの桁がずれる場合のルーラー桁合わせの扱い（SOSI 表示機能との整合）。
- ステータスバー項目の文言・アイコン・配置（既存ステータスバー項目が無いため新規追加）。
