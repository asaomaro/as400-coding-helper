## 再発パターン

### レビュー指摘・手戻りの根因（★最重要・3/3 works で共通）

**上流（research/spec）での「既存コード実装・PJ自身のスキーマ型・既存規約」の確認不足**が、下流（coding/review）で判明し、手戻りまたは decisions 是正を生んでいる。母数3件すべてで観測:

- **sndpgmmsg**: IBM 原典は読んだが **PJ自身のスキーマ型 `src/prompter/types.ts` を未確認**。「enum 欄なし」と誤認(D1)し固定値を text+help で実装 → review が dropdown 未使用を捕捉 → **review→coding 差し戻し**。
- **ruler-display**: spec「6桁目 `*`」に対し実装は既存 `rpgTabNavigation` 規約に合わせ「7桁目」を採用(D3)。既存 private `getCNewOpcodes()` の扱いも spec に明記なく coding で判断(D2)。いずれも decisions での事後是正。
- **extend-target**: `.cmd/.dds` を既存言語へ同居させる spec 方針が、**言語同居の副作用（CL誤診断・RPG編集機能の誤作動）を spec 前に未調査** → review→coding 差し戻し(D1)。※DESIGN §6 が research 工程追加の動機として記録済。

→ 症状は工程ごとに違うが、**根は同一＝「PJ内部の既存実装・型・規約という出力先仕様を上流で読み切れていない」**。外部原典(IBM docs 等)の調査はできているが、**自リポジトリ側の確認が弱い**。
