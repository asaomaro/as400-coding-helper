# aidev charter（planner の錨）

`aidev-util-propose`（planner）が課題提案する際の方針。スコープ・品質・優先度をここで縛る。
これは **PJ 側の設定ファイル**であり、汎用ハーネス（`.claude/skills/aidev-*`）はこのパスを規約として読むだけで、
中身（PJ固有のゴール・ドメイン知識）には依存しない（疎結合の継ぎ目）。

## ゴール

- VSCode 上で AS400 のコーディング（RPG/RPGLE/CLP/DDS/PRTF/CMD）を支援する拡張機能の主要機能を実装・拡充する。
  - 固定長フォーマットの入力補助（プロンプター）、桁位置ルーラー、SOSI（DBCS制御コード）表示の3本柱。

## スコープ

### 対象（提案してよい領域）

- **プロンプター定義の拡充**：CLコマンド定義 JSON（`vscode-extension/resources/prompter/cl/`）、
  RPG 仕様書定義 JSON（`vscode-extension/resources/prompter/rpg/`）の追加・改善。
- **プロンプター機能**：可変パラメータ、グルーピング、パラメータ/コマンドヘルプ、バリデーション、
  入力補助UI（dropdown 等）、定義スキーマ（`src/prompter/types.ts`）の表現力拡張。
- **ルーラー表示**：固定長の桁位置ルーラー、対象拡張子、ステータスバー切替。
- **SOSI（制御コード）表示**：DBCS 前後の `{` `}` 表示、対象拡張子、ステータスバー切替。
- **固定長編集支援**：タブナビゲーション、コメントトグル等、上記を支える編集機能。
- **横断する技術的負債の解消**（insights/retro 由来）。

### 対象外 / non-goals（提案しない）

- **フリーフォーマット（free format）RPG 対応**（AGENTS.md 明記：固定長フォーマットのみサポート）。
- **デプロイ / CI・インフラ構築**（拡張機能本体の責務外。テスト実行環境整備は別扱い）。
- **実験的な UI 刷新・SEU 互換を超える独自機能**（既存ワークフロー踏襲を優先）。
- AS400 実機（IBM i）との接続・実行・コンパイル等のランタイム連携。

## 品質基準

- 定義スキーマは **`vscode-extension/src/prompter/types.ts` に準拠**する。
- **既存挙動を壊さない**（後方互換。表示系はコードを書き換えず入力補助として重畳）。
- 仕様の根拠は **IBM 原典 / 固定長フォーマットリファレンス**を正とし、加えて
  **PJ内部の既存実装・型・既存規約を必ず確認**する（issue #13 / insights 2026-06-20 の再発根因対策）。
- **日本語ヘルプ**をコマンド/パラメータに付す。
- `tsc` 通過・桁位置検証・（環境が整えば）テスト通過。

## 優先度の指針

1. **AGENTS.md の主要機能ギャップ**（未実装の柱・コア機能）。
2. **再発する技術的負債**（`insights/*` ・ `retro.md` 由来。横断で繰り返すもの）。
3. **個別の定義追加**（CL/RPG バックログの消化）。
4. **軽微な改善 / follow-up**（nit・リファクタ）。

## 制約

- **1 PR = 独立してレビュー可能な粒度**（低結合・単独検証可。split 判定は DESIGN に従う）。
- PJ 規約（`AGENTS.md`）に従う。
- バックログ駆動の定義作成は PJ skill（CL: `cl-command-def` 等）へ委譲する前提。
- 重複起票しない（既存 open issue / `.aidev/backlog/*.md` と突き合わせる）。

## 提案の根拠にする信号（grounded 優先）

- `.aidev/insights/*` / `.aidev/works/*/retro.md` / lint・テスト・TODO・既知の負債 / 既存 issue・バックログ
