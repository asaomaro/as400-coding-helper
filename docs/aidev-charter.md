# aidev charter（planner の錨）

`aidev-propose`（planner）が課題提案する際の方針。スコープ・品質・優先度をここで縛る。
※これはテンプレート。実際のゴールに合わせて各PJが埋める。

## ゴール
- <このプロダクト/期間で達成したいこと。例: AS400 コーディング支援の主要機能を実装する>

## スコープ
### 対象（提案してよい領域）
- <例: プロンプター定義の拡充、ルーラー/SOSI表示、固定長編集支援>

### 対象外 / non-goals（提案しない）
- <例: フリーフォーマット RPG 対応、デプロイ/インフラ、実験的UI刷新>

## 品質基準
- <例: 既存スキーマ(types.ts)準拠、テスト通過、既存挙動を壊さない、日本語ヘルプ必須>

## 優先度の指針
- <例: 1) AGENTS.md の主要機能ギャップ 2) 再発する負債(insights/retro由来) 3) 軽微な改善>

## 制約
- <例: 1 PR は独立・レビュー可能な粒度、IBM原典準拠、PJ規約(AGENTS.md)に従う>

## 提案の根拠にする信号（grounded 優先）
- `.aidev/insights/*` / `.aidev/works/*/retro.md` / lint・テスト・TODO・既知の負債 / 既存 issue・バックログ
