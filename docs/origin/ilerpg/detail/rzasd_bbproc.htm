%PROC (現行プロシージャーの戻り値の名前)

Last Updated: 2021-04-13
%PROC()

%PROC は、現行プロシージャーの外部名を返します。

%PROC は、演算ステートメントでのみ指定できます。

  • サイクル・メイン・プロシージャーの場合、プロシージャーの外部名は、コンパイルされたときのモジュールの名前になります。
  • リニア・メイン・プロシージャーの場合、プロシージャーの外部名は、メイン・プロシージャーの大文字形式の名前になります。以下の例の  1  を参照してください。
  • EXTPROC が指定されていないサブプロシージャーの場合、プロシージャーの外部名は、プロシージャーの大文字形式の名前になります。以下の例の  2  を参照してください。
  • EXTPROC が指定されているサブプロシージャーの場合、プロシージャーの外部名は、EXTPROC により指定された値になります。以下の例の  3  を参照してください。
  • Java™ プロシージャーの場合、プロシージャーの外部名は、「Java_classname_methodname」形式で指定されます。以下の例の  4  を参照してください。
図 1. %PROC の例
CTL-OPT MAIN(myPgm);
DCL-S x VARCHAR(100);

DCL-PR myPgm EXTPGM('MYLIB/MYPGM345') END-PR;
DCL-PR proc1 END-PR;
DCL-PR proc2 EXTPROC('myProc2') END-PR;
DCL-PR proc3 EXTPROC(*JAVA:'MyClass':'proc3') END-PR;

DCL-PROC myPgm;  //  1 
   x = %PROC();
   // x = "MYPGM"
END-PROC;

DCL-PROC proc1;  //  2 
   x = %PROC();
   // x = "PROC1"
END-PROC;

DCL-PROC proc2;  //  3 
   x = %PROC();
   // x = "myProc2"
END-PROC;

DCL-PROC proc3 EXPORT; //  4 
   x = %PROC();
   // x = "Java_MyClass_proc3"
END-PROC;
