Delay Job (DLYJOB)

Where allowed to run: All environments (*ALL)
Threadsafe: Yes 		Parameters
Examples
Error messages

The Delay Job (DLYJOB) command causes your current job to wait for a specified number of seconds, or until a specified time of day, before running resumes.

Note: A value must be specified for either the Job delay time (DLY) parameter or the Resume job time (RSMTIME) parameter but not for both.

Job delay time (DLY)

Specifies the number of seconds to delay the job. The DLY keyword allows a maximum value of 999999 seconds (approximately 11 days, 14 hours). This is a required parameter if the Resume job time (RSMTIME) parameter is not specified.

1-999999
Specify the number of seconds to delay the job.
Top

Resume job time (RSMTIME)

Specifies the time of day the job resumes running. When specifying a resume time without specifying a resume date, the date is implied by the time specified. If the resume time is later than the current time of day, the date is assumed to be the current date. If the resume time is earlier than the current time of day, then tomorrow's date is assumed (the delay lasts until that time tomorrow). This is a required parameter if the Job delay time (DLY) parameter is not specified.

time
Specify the time of day the job resumes running.
Top

Resume job date (RSMDATE)

Specifies the date the job resumes running. This parameter can only be specified when the Resume job time (RSMTIME) parameter is specified. The resume date and resume time must be later than the current date and time.

*NEXT
The date when the next resume time occurs. If the resume time is later than the current time of day, the current date is used. If the resume time is earlier than the current time of day, then tomorrow's date is used.
date
Specify the date the job resumes running.
Top

Controlled end action (CTLEND)

Specifies the action to be taken if the job is ended in a controlled manner before the delay has completed.

*DFT
The value in environment variable QIBM_DLYJOB_CTLEND is used to determine the default action for a controlled end. If the environment variable is not defined or not set to one of the CTLEND parameter values, then *DLY is the default action.
*DLY
The delay continues until it completes or the controlled end delay time expires and the job is ended.
*END
The job is ended immediately without waiting for the delay to complete or the controlled end delay time to expire.
*RTN
Control returns to the caller of the DLYJOB command without waiting for the delay to complete.

Examples

Example 1: Delaying a Job for 15 Seconds 

DLYJOB   DLY(15)

This command delays running of the job for 15 seconds.

Example 2: Delaying a Job Until Midnight 

DLYJOB   RSMTIME('000000')

This command delays running of the job until midnight tonight.

Example 3: Delaying Until Future Date or Controlled End 

DLYJOB   RSMTIME('08:00:00')
         RSMDATE('06/01/36')
         CTLEND(*RTN)
RTVJOBA   ENDSTS(&JOBEND)

This command delays running of the job until June first in the year 2036 or until the job is ended controlled. The RTVJOBA command gets the controlled end status.
