DLTEDT (Delete Edit) keyword for display files

Last Updated: 2026-07-13

You use this field-level keyword to specify that the IBM i operating system is to ignore the EDTCDE or EDTWRD keyword if either of them is specified for a referenced field. This keyword is valid only when you specify R in position 29.

This keyword has no parameters.

If you specify a new editing keyword, DLTEDT is unnecessary. The new editing keyword overrides the referenced editing keyword.

Option indicators are not valid for this keyword.

Example

The following example shows how to specify the DLTEDT keyword.

|...+....1....+....2....+....3....+....4....+....5....+....6....+....7....+....8
00040A                                      REF(FILEA)
00050A          R RECORD
00060A            AMT       R        B  5 20DLTEDT
     A
