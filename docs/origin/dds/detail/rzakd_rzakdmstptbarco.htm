印刷装置ファイルの BARCODE (バーコード) キーワード

Last Updated: 2021-04-13

このフィールド・レベルのキーワードは、 フィールドをユーザー指定のバーコードとして印刷する場合に使用します。 BARCODE キーワードは、高機能印刷装置データ・ストリーム (IPDS) プリンターおよびホスト印刷変換プリンターの場合にのみ有効です。また、このキーワードは、装置タイプ *IPDS または *AFPDS が指定された印刷装置ファイルの場合にのみ有効です。

キーワードの形式は次のとおりです。 
BARCODE(bar-code-ID [height] [[*HRZ | *VRT]
        [*HRI | *HRITOP | *NOHRI] [*AST | *NOAST]
        [modifier] [unit-width] [symbol-width]
        [wide/narrow-ratio]
        [PDF417 data]
        [Data Matrix data]
        [Maxicode data]
        [QR Code data]])

bar-code-ID は必須パラメーターです。bar-code-ID パラメーターは、10 文字までの特殊英数字、または数値 ID として指定できます。bar-code-ID として有効な特殊値は表 1 に示すとおりです。 bar-code-ID に数値を指定する場合は、バーコード修飾子パラメーターも指定する必要があります。 ご使用のプリンターがサポートする bar-code-ID と修飾子については、当該プリンターの資料を参照してください。

height パラメーターの指定は任意ですが、これを指定する場合は、キーワードに 続く 2 番目のパラメーターとしなければなりません。この高さは、次のいずれかの形式で指定することができます。
  • 印刷行数単位で高さを指定する。バーコードの高さとして有効な値は 1 から 9 までです。
  • インチまたは cm 単位で高さを指定する。この方法で高さを指定する場合、有効な形式は (height *UOM) です。 有効な値は、0.25 から 25.40 cm (0.10 から 10.00 インチ) です。 PDF417 バーコード・タイプの場合、有効な値は 0.02 から 25.40 cm (0.01 から 10.00 インチ) です。 印刷装置ファイルの作成 (CRTPRTF) コマンドの UOM パラメーターが、高さの計測単位を決定します。

バーコードの高さとして指定する値には、バーコードの下の人間の目で読める解釈値は含まれていません。 height パラメーター を指定しなかった場合、プリンターではデフォルトの高さが使用されます。

注: バーコードによっては、記号によって高さが指定されるため、ここで指定した高さが無視されるものもあります。 GS1 Databar バーコードなどその他のバーコードについては、修飾子によっては、バーコードの高さを、少なくともモジュール幅の一定乗数と同じ高さにしなければならないことがあります。 それらの場合は、高さはその記号の要件に合うように調整されます。

最後の 11 個のパラメーターはすべて任意指定であり、かつどのような順序で指定してもかまいません。 これらのパラメーターを使用すれば、BARCODE キーワードによって 次のことを指定することができます。

  • バーコードを横方向また縦方向に印刷する。デフォルトは横方向印刷 (*HRZ) です。
  • 人間の目で読めるバーコードの解釈値を組み込むか、または除外する。デフォルトでは、バーコードの下に人間の目で読める解釈値を組み込みます (*HRI)。
  • 人間の目で読める解釈値がバーコードの上に印刷されることを示す (*HRITOP)。 (*HRITOP のバーコード・サポートはプリンターによって異なるので、個々の プリンターの解説書を参照してください。)
  • 人間の目で読める CODE3OF9 バーコードの解釈値を囲むアスタリスクを組み込むか、または除外する。 デフォルトはアスタリスクの 除外 (*NOAST) です。
  • バーコード修飾子を選択する。 この修飾子は 16 進数 FF 以外の 1 文字の 16 進数値です。
  • 幅の狭いバーコードの幅 (インチ) を指定する。これは、(*WIDTH 値) の形式で指定します。式を指定する方法についての詳細は、「DDS 解説書: 概念」を参照してください。このパラメーターに指定できる値は、0.007 から 0.208 です。
  • 幅の狭いバーコードに対する広いバーコードの比率を指定する。これは、(*RATIO 値) の形式で指定します。このパラメーターに指定できる値は、2.00 から 3.00 です。
  • バーコードの要求記号幅をインチで指定する。 これは、形式 (*SWIDTH value) を表すために指定されます。 このパラメーターをサポートするプリンターは、指定された要求記号幅以内に収まる最大幅のバーコード記号を作成しようと試みます。 これらのプリンターではモジュール幅は無視されます。 有効な値は 0.00 から 9.99 です。 ゼロの値が指定されたり、このパラメーターが指定されなかったりした場合は、要求記号幅が無視され、モジュール幅がバーコード・サイズを決めるために使用されます。
  • 2 次元バーコード PDF417、Maxicode、Data Matrix、および QR Code の追加バーコード情報を指定する。 これらの 2 次元バーコード・データ値のうちいずれか 1 つのみを指定することができます。 詳しくは、2 次元バーコードの指定を参照してください。
注: バーコードの全体的な幅は以下の要素によって決まります。
  • ユーザー DDS の比率パラメーターまたは幅パラメーター
  • ユーザー DDS 内の要求記号幅
  • バーコード内の実際の顧客データ
  • プリンター・ハードウェアの制限 (PEL 密度、ピンなど)

指定した bar-code-ID に当てはまらない任意指定パラメーターを指定した場合、プリンターはその任意指定パラメーターを無視します。

バーコードを印刷する機能のないプリンターでバーコードを印刷しようとすると、コードの数字はテキストとして扱われ、バーコードは印刷できないということ を示す診断メッセージが表示されます。

該当のフィールドに対して指定する行および桁は、バーコードの左上隅の位置として 使用されます。DDS で指定された行は基線 (その上に文字が印刷される仮想上の線) であり、この線はバーコードの上端として使用されるので、バーコードは指定した行の底部から下方へ伸びるように現れます。

  1. 4234 プリンターは 14 桁までしかサポートしません。
  2. バーコード修飾子の値 01 は、ある種のプリンターでは無効です。
  3. 値 01 は、AFP フォントを使って日本の郵便番号を印刷する アプリケーション・プログラムの移行サポートを提供します。フィールドに 書き込まれるデータは、AFP フォントで有効な文字でなければなりません。 また、アプリケーション・プログラムは開始文字、停止文字、およびバーコード修飾文字 を書き込むことも必要です。
  4. PDF417 の場合、セキュリティー・レベルに応じて、記号ごとに最高 1850 文字のテキスト文字、または 1108 バイトのバイナリー・データ。記号の仕様を参照してください。 Maxicode の場合、記号ごとに最高 93 文字の英数字 (エンコードによる) または記号ごとに最高 138 文字の数字。記号の仕様を参照してください。 Data Matrix の場合、データが文字であるか数字であるかに応じて、最高 3116 文字。記号の仕様を参照してください。 QR Code の場合、データが文字であるか数字であるかに応じて、最高 7089 文字。記号の仕様を参照してください。
  5. POSTNET bar-code-ID およびバーコード修飾子 04 を指定すると PLANET バーコードが選択されます。
  6. CODE128 bar-code-ID およびバーコード修飾子 03 または 04 を指定すると、UCC/EAN 128 (または GS1-128) バーコードを選択することができます。 CODE128 bar-code-ID およびバーコード修飾子 05 を指定すると、USPS Intelligent Mail Container バーコードを選択することができます。
  7. 修飾子が異なれば、許容文字数および有効文字のセットに対する要件も異なります。 有効な値を確認するには、プリンター資料または各バーコードの記号仕様を参照してください。
  8. INTERL2OF5 bar-code-ID およびバーコード修飾子 03 または 04 を指定すると、GS1 ITF-14 バーコード (ベアラーバー付きのインターリーブド 2 of 5) を選択することができます。

CODEABAR フィールドのデータは、A、B、C、または D で始まり、A、B、C、ま たは D で終わっていなければなりません。例えば、A11224455C や D33447799D などです。

BARCODE は、CHRSIZ、CHRID、CVTDTA、DATE、EDTCDE、EDTWRD、FONT、HIGHLIGHT、 PAGNBR、TIME、または UNDERLINE キーワードと同じフィールドに指定してはなりません。

IPDS バーコード・コマンドのコーディングについては、CVTDTA キーワードを参照してください。

CHRSIZ キーワードをレコード・レベルで指定した場合、それはそのレコード様式内 のすべてのフィールドに適用されます。これらのフィールドのどれか で BARCODE キーワードを使用することはできません。

同じレコード様式で BLKFOLD、CPI、または DFNCHR キーワードと BARCODE キーワードを同時に指定することはできません。

数字フィールドについて BARCODE を指定する場合には、小数点以下の桁数はゼロでなければなりません。

固定情報フィールドに BARCODE を指定するときは、有効な bar-code-ID は CODEABAR、CODE128、および CODE3OF9 のみであり、DFT キーワードも暗黙または明示のいずれかで指定する必要があります。

ファイル内で BARCODE が指定されたときは、CRTPRTF コマンドで DEVTYPE(*IPDS) または DEVTYPE(*AFPDS) を指定してください。

BARCODE はデータ・タイプ S および A でしか使用できません (制約事項については表 2 を参照)。ただし、O および G データ・タイプでも使用できる QR Code バーコードを除きます。

オプション標識は、このキーワードでは無効です。

日本の郵便番号 (bar-code-ID = JPBC) は、bar-code-ID パラメーター、バーコード印刷方向パラメーター ([*HRZ | *VRT])、およびバーコード修飾子パラメーターだけを使用します。それ以外のすべてのパラメーターには 事前に定義された値があるので、パラメーターに入力しても無視されます。

ユーザー指定のバーコード修飾子は、妥当性を検査されないため、バーコード修飾子が無効な場合には、バーコード・エラーが起こることがあります。バーコードと有効なバーコード修飾子について詳しくは、「Intelligent Printer Data Stream Reference」を参照してください。

2 次元バーコードの指定

2 次元バーコード PDF417、Maxicode、Data Matrix、および QR Code の追加パラメーターを指定することができます。

PDF417: 追加データは次の形式で指定します。 
(*PDF417 row-size number-rows security 
         [escape-indicator] [*MACRO(&data-field-name)]
         [convert-indicator] [trim-indicator])
row-size
これは PDF417 バーコードの必須パラメーターであり、1 行当たりのデータ記号文字数を指定します。 有効な値は 1 から 30 です。
number-rows
これは PDF417 バーコードの必須パラメーターであり、行数を指定します。 有効な値は 3 から 90 です。特殊値 *MIN (minimum) を指定すると、最小限必要な行数を生成するようにプリンターに指示できます。
security
これは PDF417 バーコードの必須パラメーターであり、セキュリティー・レベルを指定します。 有効な値は 0 から 8 です。セキュリティー・レベルが高くなるごとに、記号に追加される誤り訂正コード・ワード数が増えます。特定のセキュリティー・レベルでは、いくつかのコード・ワードが欠落または消去されている場合がありますが、記号は回復できます。
escape-indicator
これは PDF417 バーコードの任意指定パラメーターであり、バーコード・データ内のバックスラッシュ文字が PDF417 記号仕様に従ってエスケープ文字とみなされるかどうかを指定します。 エスケープ文字 (バックスラッシュで始まる) により、バーコード・データ内でリーダー・プログラミングを指定することができます。 escape-indicator の有効な値は、次のとおりです。
  • *NOESCAPE は、バーコード・データ内の各バックスラッシュ文字が通常のデータ文字とみなされるので、すべてのエスケープ・シーケンスが無視されることを示します。 バーコード・データがイメージ・データまたはバイナリー・データである場合は、*NOESCAPE を指定してください。 escape indicator が指定されない場合、*NOESCAPE がデフォルトです。
  • *ESCAPE は、バーコード・データ内の各バックスラッシュ文字が PDF417 記号仕様に従ってエスケープ文字とみなされることを示します。
*MACRO(&data-field-name)
この任意指定パラメーターは、PDF417 Control Block コーディングを指定できるようにします (Uniform Symbology Specification PDF417 の G.2 項で定義)。 マクロ・データは、プログラム - システム間フィールドとして指定する必要があります。このフィールドは、BARCODE キーワードと同じレコード様式内に存在しなければなりません。マクロ・データの長さは、レコード様式の規則によって制限されます。つまり、1 つのレコード様式内の名前のあるすべてのフィールドと標識の桁数を合計した最大値は、32 767 バイトです。データ・タイプは A (文字)、使用目的は P (プログラム - システム間) でなければなりません。
convert-indicator
これは PDF417 バーコードの任意指定パラメーターであり、通常の EBCDIC から ASCII へのデータ変換が行われないことを指定します。 convert-indicator の値としては *NOCONVERT のみが有効です。 *NOCONVERT が指定されると、データはデフォルト・エンコード (ASCII) であるとみなされ、バーコードが作成されるまで変換されません。 このパラメーターが指定されないと、通常の EBCDIC から ASCII への変換が行われます。 変換について詳しくは、2 次元バーコードの EBCDIC から ASCII への変換を参照してください。
trim-indicator
これは PDF417 バーコードの任意指定パラメーターであり、バーコード・データ内の末尾ブランクを切り取って、それがプリンターに送られないようにすることを指定します。 trim-indicator の値としては *TRIM のみが有効です。 *TRIM が指定されると、データ内にある最後の非ブランク文字の後に続くブランクはすべて除去され、プリンターに渡されません。 このパラメーターが指定されないと、末尾ブランクは除去されません。 *NOCONVERT が *TRIM と一緒に指定された場合は、末尾の ASCII ブランク (X'20') が切り取られます。 そうでない場合は、末尾の EBCDIC ブランク (X'40') が切り取られます。
Maxicode: 追加データは次の形式で指定します。 
(*MAXICODE symbol-mode [zipper-indicator] [escape-indicator]
           [sequence-data] [convert-indicator] [trim-indicator])
symbol-mode
これは Maxicode バーコードの必須パラメーターであり、MaxiCode バーコードの symbol-mode を指定します。 有効な値は 2 から 6 です。
  • 2 – 構造化キャリア・メッセージ、数値の郵便番号
  • 3 – 構造化キャリア・メッセージ、英数字の郵便番号
  • 4 – 標準コード
  • 5 – 完全 ECC 記号
  • 6 – リーダー・プログラム、SEC。 データは送信されません。
zipper-indicator
これは Maxicode バーコードの任意指定パラメーターであり、ジッパー・パターンおよびコントラスト・ブロックを印刷するかどうかを指定します。 zipper-indicator の有効な値は次のとおりです。
  • *NOZIPPER は、ジッパー・パターンがバーコードと一緒に印刷されないことを示します。 zipper-indicator が指定されない場合、*NOZIPPER がデフォルトです。
  • *ZIPPER は、ジッパー・パターンがバーコードと一緒に印刷されることを示します。
escape-indicator
この任意指定パラメーターは、バーコード・データ内のバックスラッシュ文字が Maxicode 記号仕様に従ってエスケープ文字とみなされるかどうかを指定します。 エスケープ文字 (バックスラッシュで始まる) により、バーコード・データ内でリーダー・プログラミングを指定することができます。 escape-indicator の有効な値は、次のとおりです。
  • *NOESCAPE は、バーコード・データ内の各バックスラッシュ文字が通常のデータ文字とみなされるので、すべてのエスケープ・シーケンスが無視されることを示します。 バーコード・データがイメージ・データまたはバイナリー・データである場合は、*NOESCAPE を指定してください。 escape indicator が指定されない場合、*NOESCAPE がデフォルトです。
  • *ESCAPE は、バーコード・データ内の各バックスラッシュ文字が Maxicode 記号仕様に従ってエスケープ文字とみなされることを示します。
sequence-indicator
これは Maxicode バーコードの任意指定パラメーターであり、このバーコードが構造化された付加の一部であるかどうかを指定します。 Maxicode バーコードは、大量のデータをエンコードするために論理的に連結することができます。 論理的にリンクされた記号は、同じメディアまたは異なるメディア上に提示でき、スキャンされた後に論理的に再結合されます。 Maxicode の順序データは、次の形式の 2 つの部分から構成されます。 
*SEQUENCE(sequence-indicator total-symbols)

sequence-indicator は、構造化された付加順序標識を指定します。有効値は 0 から 8 です。

total-symbols は、構造化された付加記号の合計数を指定します。 有効値は 2 から 8 です。

convert-indicator
これは Maxicode バーコードの任意指定パラメーターであり、通常の EBCDIC から ASCII へのデータ変換を行わないことを指定します。 convert-indicator の値としては *NOCONVERT のみが有効です。 *NOCONVERT が指定されると、データはデフォルト・エンコード (ASCII) であるとみなされ、バーコードが作成されるまで変換されません。 このパラメーターが指定されないと、通常の EBCDIC から ASCII への変換が行われます。 変換について詳しくは、2 次元バーコードの EBCDIC から ASCII への変換を参照してください。
trim-indicator
これは Maxicode バーコードの任意指定パラメーターであり、バーコード・データ内の末尾ブランクを切り取って、それがプリンターに送られないようにすることを指定します。 trim-indicator の値としては *TRIM のみが有効です。 *TRIM が指定されると、データ内にある最後の非ブランク文字の後に続くブランクはすべて除去され、プリンターに渡されません。 このパラメーターが指定されないと、末尾ブランクは除去されません。 *NOCONVERT が *TRIM と一緒に指定された場合は、末尾の ASCII ブランク (X'20') が切り取られます。 そうでない場合は、末尾の EBCDIC ブランク (X'40') が切り取られます。
Data Matrix: 追加データは次の形式で指定します。 
(*DATAMATRIX row-size number-rows [alternate-data-type] [reader] 
       [header-trailer] [escape-indicator] [sequence-indicator]
       [convert-indicator] [encoding-scheme] [trim-indicator])
Data Matrix バーコードでは、以下のパラメーターを示されている順序で指定することができます。
row-size
これは Data Matrix バーコードの必須パラメーターであり、行サイズを指定します。 Data Matrix バーコードは正方形記号または長方形記号です。 正方形の記号は 10 x 10 から 144 x 144 であり、長方形の記号は 8 x 18 か ら 16 x 48 です。特殊値 *DFT を指定すると、プリンターに記号データの量に基づいて行サイズを選択させることができます。
number-rows
これは Data Matrix バーコードの必須パラメーターであり、行数を指定します。 Data Matrix バーコードは正方形記号または長方形記号です。 正方形の記号は 10 x 10 から 144 x 144 であり、長方形の記号は 8 x 18 か ら 16 x 48 です。特殊値 *DFT を指定すると、プリンターに記号データの量に基づいて行数を選択させることができます。
alternate-data-type
これは Data Matrix バーコードの任意指定パラメーターであり、定義された記号のデータ・タイプを指定します。 有効な値は次のとおりです。
  • *USRDEF は、その記号がユーザー定義の記号であることを示します。 alternate-data-type が指定されない場合、*USRDEF がデフォルトです。
  • *AIMSTD は、記号が、AIM international によって認定された特定の業界標準に準拠することを指定します。
  • *UCCEAN は、記号が UCC/EAN アプリケーション ID 標準形式に準拠することを指定します。
reader
これは Data Matrix バーコードの任意指定パラメーターであり、このバーコードがリーダー・システムのプログラミングに使用されるメッセージをエンコードするかどうかを指定します。 有効な値は次のとおりです。
  • *DATA は、バーコード・データが提供されることを示します。 reader 標識が指定されない場合、*DATA がデフォルトです。
  • *RDRPRG は、バーコード・リーダーのプログラミングに使用されるメッセージが記号に含まれていることを示します。
header-trailer
これは Data Matrix バーコードの任意指定パラメーターであり、バーコード・リーダーに対するヘッダーとトレーラーの指示が含まれるかどうかを指定します。 有効な値は次のとおりです。
  • *NO は、ヘッダーもトレーラーも挿入されないことを指定します。 header-trailer 値が指定されない場合、*NO がデフォルトです。
  • *HEADER5 は、バーコード・リーダーが 05 Macro コード・ワードを挿入することを示します。
  • *HEADER6 は、バーコード・リーダーが 06 Macro コード・ワードを挿入することを示します。
escape-indicator
この任意指定パラメーターは、バーコード・データ内のバックスラッシュ文字が Data Matrix 記号仕様に従ってエスケープ文字とみなされるかどうかを指定します。 エスケープ文字 (バックスラッシュで始まる) により、バーコード・データ内でリーダー・プログラミングを指定することができます。 escape-indicator の有効な値は、次のとおりです。
  • *NOESCAPE は、バーコード・データ内の各バックスラッシュ文字が通常のデータ文字とみなされるので、すべてのエスケープ・シーケンスが無視されることを示します。 バーコード・データがイメージ・データまたはバイナリー・データである場合は、*NOESCAPE を指定してください。 escape indicator が指定されない場合、*NOESCAPE がデフォルトです。
  • *ESCAPE は、バーコード・データ内の各バックスラッシュ文字が Data Matrix 記号仕様に従ってエスケープ文字とみなされることを示します。
sequence-indicator
これは Data Matrix バーコードの任意指定パラメーターであり、このバーコードが構造化された付加の一部であるかどうかを指定します。 Data Matrix バーコードは、大量のデータをエンコードするために論理的に連結することができます。 論理的にリンクされた記号は、同じメディアまたは異なるメディア上に提示でき、スキャンされた後に論理的に再結合されます。 Data Matrix の順序データは、以下の形式の 3 つの部分から構成されます。 
*SEQUENCE(sequence-indicator total-symbols file-id)

sequence-indicator は、構造化された付加順序標識を指定します。有効な値は 1 から 16 です。

total-symbols は、記号の順序で論理的にリンクされる記号の合計数を指定します。有効な値は 2 から 16 です。

file-id は、一組の構造化付加記号に対する 2 バイトの固有のファイル ID を指定します。 file-id を指定する際の形式は X'nnnn' です。ここで、nnnn は 2 バイトのファイル ID (16 進表記) です。

convert-indicator
これは Data Matrix バーコードの任意指定パラメーターであり、通常の EBCDIC から ASCII へのデータ変換を行わないことを指定します。 convert-indicator の値としては *NOCONVERT のみが有効です。 *NOCONVERT が指定されると、データはデフォルト・エンコード (ASCII) であるとみなされ、バーコードが作成されるまで変換されません。 このパラメーターが指定されないと、通常の EBCDIC から ASCII への変換が行われます。 変換について詳しくは、2 次元バーコードの EBCDIC から ASCII への変換を参照してください。
encoding-scheme
これは Data Matrix バーコードの任意指定パラメーターであり、バーコード記号を生成する際に使用されるエンコード・スキームを指定します。 有効な値は次のとおりです。
  • *DEVDFT - このエンコード・スキームは、エンコード・スキームの選択および切り替えに際して BCOCA レシーバーが装置に固有の方式を使用することを示します。 *DEVDFT は、encoding-scheme が指定されなかった場合のデフォルトです。
  • *ASCII - このエンコード・スキームは、2 桁の数値の場合は 1 データ文字につき 4 ビットを、ASCII 値 0 から 127 の場合は 1 データ文字につき 8 ビットを、拡張 ASCII 値 128 から 255 の場合は 1 データ文字につき 16 ビットを、それぞれ生成します。
  • *C40 - このエンコード・スキームは、入力データが主に大文字の英数字であるときに使用され、1 データ文字につき 5.33 ビットを生成します。
  • *TEXT - このエンコード・スキームは、入力データが主に小文字の英数字であるときに使用され、1 データ文字につき 5.33 ビットを生成します。
  • *X12 - このエンコード・スキームは、入力データが ANSI X12 EDI データ・セットで定義されたときに使用され、1 データ文字につき 5.33 ビットを生成します。
  • *EDIFACT - このエンコード・スキームは、入力データが ASCII 値 32 から 94 であるときに使用され、1 データ文字につき 6 ビットを生成します。
  • *BASE256 - このエンコード・スキームは、入力データがバイナリー (例えば、イメージ・データや非テキスト・データなど) であるときに使用され、1 データ文字につき 8 ビットを生成します。
  • *AUTO - このエンコード・スキームは、BCOCA レシーバーが ASCII エンコードから始めて、生成される記号データ文字を最小限に抑えるために必要に応じてエンコード・スキームを切り替えることを示します。
trim-indicator
これは Data Matrix バーコードの任意指定パラメーターであり、バーコード・データ内の末尾ブランクを切り取って、それがプリンターに送られないようにすることを指定します。 trim-indicator の値としては *TRIM のみが有効です。 *TRIM が指定されると、データ内にある最後の非ブランク文字の後に続くブランクはすべて除去され、プリンターに渡されません。 このパラメーターが指定されないと、末尾ブランクは除去されません。 *NOCONVERT が *TRIM と一緒に指定された場合は、末尾の ASCII ブランク (X'20') が切り取られます。 そうでない場合は、末尾の EBCDIC ブランク (X'40') が切り取られます。
注: 構造化付加が指定される場合、header と trailer に *NO を指定 し、reader に *DATA を指定する必要があります。 row-size と number-rows の有効な組み合わせについては、記号の仕様を参照してください。
QR Code: 追加データは次の形式で指定します。 
(*QRCODE verson error-correction [alternate-data-type]
         [escape-indicator] [sequence-indicator]
         [convert-indicator] [trim-indicator])
QR Code バーコードでは、以下のパラメーターを示されている順序で指定することができます。
version
これは QR Code バーコードの必須パラメーターであり、記号の要求サイズを指定します。各 version は、行および列ごとの特定数のモジュールを、モジュール幅パラメーターによって決定される各正方形モジュールのサイズで指定します。 有効な値は 0 から 40 です。 このパラメーターに 0 が指定された場合は、記号データの量に基づいて適切な行/列サイズが使用され、当該データ量を収容できる最小の記号が生成されます。
error-correction
これは QR Code バーコードの必須パラメーターであり、当該記号に対して使用される誤り訂正レベルを指定します。 誤り訂正レベルが高くなるごとに記号に追加される誤り訂正コード・ワードが増え、したがって、その分、記号データ用のコード・ワードが減少することになります。 version と error-correction レベルの組み合わせごとに記号データで使用可能なコード・ワード数について詳しくは、QR Code 記号仕様を参照してください。 次に示す 4 つの異なるレベルのリード・ソロモン誤り訂正が選択可能です。
  • 0 - レベル L は記号コード・ワードの 7% を回復させます
  • 1 - レベル M は記号コード・ワードの 15% を回復させます
  • 2 - レベル Q は記号コード・ワードの 25% を回復させます
  • 3 - レベル H は記号コード・ワードの 30% を回復させます
alternate-data-type
これは QR Code バーコードの任意指定パラメーターであり、定義された記号のデータ・タイプを指定します。 有効な値は次のとおりです。
  • *USRDEF は、この記号がユーザー定義の記号であることを示します。 alternate-data-type が指定されない場合、*USRDEF がデフォルトです。
  • *AIMSTD (application-indicator) は、記号が AIM international によって認定された特定の業界標準またはアプリケーション標準に準拠することを示します。 application-indicator は 0 から 255 の数値です。
  • *UCCEAN は、記号が UCC/EAN アプリケーション ID 標準形式に準拠することを指定します。
escape-indicator
これは QR Code バーコードの任意指定パラメーターであり、バーコード・データ内の X'5C'(¥) 文字が QR Code 記号仕様に従ってエスケープ文字とみなされるかどうかを指定します。 エスケープ文字により、バーコード・データ内でリーダー・プログラミングを指定することができます。 escape-indicator の有効な値は、次のとおりです。
  • *NOESCAPE は、バーコード・データ内の各 X'5C' (¥) 文字が通常のデータ文字とみなされるので、すべてのエスケープ・シーケンスが無視されることを示します。 バーコード・データがイメージ・データまたはバイナリー・データである場合は、*NOESCAPE を指定してください。 escape indicator が指定されない場合、*NOESCAPE がデフォルトです。
  • *ESCAPE は、バーコード・データ内の各 X'5C' (¥) 文字が QR Code 記号仕様に従ってエスケープ文字とみなされることを示します。
sequence-indicator
これは QR Code バーコードの任意指定パラメーターであり、このバーコードが構造化された付加の一部であるかどうかを指定します。 QR Code バーコードは、大量のデータをエンコードするために論理的に連結することができます。 論理的にリンクされた記号は、同じメディアまたは異なるメディア上に提示でき、スキャンされた後に論理的に再結合されます。 QR Code の順序データは、次の形式の 3 つの部分から構成されます。 
*SEQUENCE(sequence-indicator total-symbols parity-data)

sequence-indicator は、構造化された付加順序標識を指定します。有効な値は 1 から 16 です。

total-symbols は、記号の順序で論理的にリンクされる記号の合計数を指定します。有効な値は 2 から 16 です。

parity-data は、構造化付加記号のパリティー・データを指定します。 parity-data 値は、構造化付加記号に分解されるメッセージ全体から計算する必要があります。parity-data 値は各構造化付加記号で同じものになるようにしてください。 有効な値は 0 から 255 の範囲内の値です。

convert-indicator
これは QR Code バーコードの任意指定パラメーターであり、BCOCA レシーバーがデータを EBCDIC から ASCII に変換するか、データが既に ASCII であるかを指定します。 convert-indicator の有効な値は次のとおりです。
  • *NOCONVERT - データはデフォルト・エンコード (ASCII) であるとみなされ、バーコードが作成されるまで変換されません。
  • *CONVERT(conversion-type) - データは、バーコード記号が生成される前に EBCDIC から ASCII に変換されます。 使用される変換は、指定された conversion-type 値に基づいて決まります。 conversion-type の有効な値は 1 から 9 です。タイプ 1 から 3 の場合は、この変換をプリンターが行います。 IPDS 印刷装置がサポートするタイプはこれらのみです。 タイプ 4 から 9 の場合は、データがプリンターに送られる前に、ASCII への変換をシステムが行います。 変換について詳しくは、2 次元バーコードの EBCDIC から ASCII への変換を参照してください。 値とその意味は次のとおりです。
    • 1 - EBCDIC コード・ページ 500 から ASCII に変換します
    • 2 - EBCDIC コード・ページ 290 から ASCII に変換します
    • 3 - EBCDIC コード・ページ 1027 から ASCII に変換します
    • 4 - EBCDIC CCSID 5026 から ASCII CCSID 943 に変換します
    • 5 - EBCDIC CCSID 1399 から ASCII CCSID 943 に変換します
    • 6 - EBCDIC CCSID 5026 から ASCII CCSID 932 に変換します
    • 7 - EBCDIC CCSID 1399 から ASCII CCSID 932 に変換します
    • 8 - EBCDIC CCSID 5026 から ASCII CCSID 942 に変換します
    • 9 - EBCDIC CCSID 1399 から ASCII CCSID 942 に変換します
このパラメーターが指定されないと、あたかも *CONVERT(1) が指定されたかのように、プリンターは記号を生成する前に EBCDIC コード・ページ 500 から ASCII への変換を行います。
trim-indicator
これは QR Code バーコードの任意指定パラメーターであり、バーコード・データ内の末尾ブランクを切り取って、それがプリンターに送られないようにすることを指定します。 trim-indicator の値としては *TRIM のみが有効です。 *TRIM が指定されると、データ内にある最後の非ブランク文字の後に続くブランクはすべて除去され、プリンターに渡されません。 このパラメーターが指定されないと、末尾ブランクは除去されません。 *NOCONVERT が *TRIM と一緒に指定された場合は、末尾の ASCII ブランク (X'20') が切り取られます。 そうでない場合は、末尾の EBCDIC ブランク (X'40') が切り取られます。

2 次元バーコードの EBCDIC から ASCII への変換

2 次元バーコード PDF417、Maxicode、Data Matrix、および QR Code は ASCII バーコードです。 これらのバーコードに指定された convert-indicator が変換を行わないことを示す場合を除いて、システムはデータを予期された EBCDIC または ASCII コード・ページに変換します。 この変換を行うために、システムは、スプール・ファイルを生成したジョブの CCSID を抽出し、PDF417、Maxicode、および Data Matrix バーコードの EBCDIC データをコード・ページ 500 に変換します。 そうすると、プリンターは、バーコード記号を生成する前に、データをコード・ページ 500 から適切な ASCII コード・ページに変換します。

QR Code バーコードの場合は、指定された変換タイプに応じて、システムはデータをさまざまなコード・ページに変換します。 変換タイプ 1 から 3 の場合は、システムはデータをジョブの CCSID から EBCDIC コード・ページ 500、290、または 1027 に変換します。 そうすると、プリンターは、バーコード記号が生成される前に、データをそれらの予期された EBCDIC コード・ページから適切な ASCII コード・ページに変換します。 変換タイプ 4 から 9 の場合は、システムは、データが EBCDIC CCSID 5026 または EBCDIC CCSID 1399 のいずれかであるとみなし、データを ASCII CCSID 943、932、または 942 に変換します。 プリンターは、バーコード記号を生成する前、これ以上の変換を行いません。

プリンターがバーコード記号を作成すると、Maxicode および Data Matrix バーコードは 8859-1 コード・ページであるとみなされます。 IBM® で相当するものは、ASCII コード・ページ 819 です。PDF417 の場合は ASCII コード・ページ 437 が使用され、QR Code 変換タイプ 1 から 3 の場合は ASCII コード・ページ 897 が使用されます。
